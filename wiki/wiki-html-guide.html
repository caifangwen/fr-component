设计好 CSS 后，怎么写 HTML？

核心原则：先写语义结构，再加 class，最后映射到 CSS 变量。 不要反过来——不要先想"这个 div 应该怎么美化"。

步骤 1：语义化骨架（不要管样式）

先忽略视觉，只考虑内容含义。这是 HTML 的灵魂。

<!-- ✅ 好：语义清晰，无障碍友好 -->
<article>
  <header>
    <h2>文章标题</h2>
    <time datetime="2026-05-08">2026年5月8日</time>
  </header>
  <section>
    <p>正文内容...</p>
  </section>
  <footer>
    <button>点赞</button>
    <button>分享</button>
  </footer>
</article>

<!-- ❌ 坏：全是 div，没有语义 -->
<div class="wrapper">
  <div class="top">
    <div class="title">文章标题</div>
    <div class="date">2026-05-08</div>
  </div>
  <div class="body">
    <div class="text">正文内容...</div>
  </div>
</div>
为什么语义化如此重要？
  • SEO：搜索引擎通过 <article>、<time> 理解内容结构
  • 无障碍：屏幕阅读器依赖语义标签导航
  • AI 理解：AI 处理语义化 HTML 的准确率远高于纯 div 堆砌
  • 维护性：6 个月后回看代码，语义标签一眼就知道这块是什么

步骤 2：加 class，对接 CSS 变量系统

class 命名采用"组件 + 变体"策略（BEM 命名法）：

<article class="card">
  <header class="card__header">
    <h2 class="card__title">文章标题</h2>
    <time class="card__meta">2026-05-08</time>
  </header>
  <section class="card__body">
    <p>正文内容...</p>
  </section>
  <footer class="card__footer">
    <button class="btn btn--primary">点赞</button>
    <button class="btn btn--ghost">分享</button>
  </footer>
</article>

对应的 CSS（使用你的设计令牌）：

.card {
  background: var(--color-surface);
  border: 1px solid var(--color-border);
  border-radius: var(--radius-lg);
  padding: var(--space-6);
}

.card__title {
  color: var(--color-text);
  font-size: var(--text-lg);
}

.btn {
  padding: var(--space-2) var(--space-4);
  border-radius: var(--radius-md);
  border: none;
  cursor: pointer;
}

.btn--primary {
  background: var(--color-primary);
  color: #fff;
}

.btn--ghost {
  background: transparent;
  border: 1px solid var(--color-primary);
  color: var(--color-primary);
}

步骤 3：Utility Class 处理微调

对于一次性的边距、对齐等微调，不用新建组件，直接用 utility：

<div class="card mt-4 text-center">
  <!-- mt-4 = margin-top: var(--space-4) -->
</div>

三种命名方式对比

命名法示例适用场景
BEM.card__title--large中型项目，团队协作，防止命名冲突
Semantic.article-header内容型网站，可读性优先
Utility.flex .gap-2 .p-4快速原型，配合 Tailwind 使用
建议

原生三件套项目用 BEM + 少量 Utility 是最佳平衡。既有组件边界，又能快速微调。

纯 CSS + HTML + JS 落后吗？

完全不落后。 甚至在 2026 年，原生三件套正在强势复兴。

原生 vs 框架对比

维度原生三件套React / Vue 框架
加载性能零依赖，首屏毫秒级需加载运行时（40KB+）
构建复杂度保存即刷新，无 node_modules需要 Webpack / Vite 构建
长期维护10 年后照样打开，不担心版本过期依赖升级可能引发连锁问题
SEO完美，爬虫直接读内容需要 SSR / SSG 额外处理
部署直接丢到服务器或 CDN需要构建流水线
AI 辅助AI 对原生代码理解最准确框架语法糖可能让 AI 混淆

原生三件套适合哪些场景

  • ✓ 内容型网站：博客、文档、Wiki、企业官网（你现在的项目就是）
  • ✓ WordPress 主题：PHP 输出 HTML + 自定义 CSS，最稳定
  • ✓ 快速原型：验证想法时，原生比搭框架快 10 倍
  • ✓ 长期存档：知识库、简历页、作品展示，要求 10 年后还能打开
  • ✓ 嵌入式页面：营销活动页、落地页，要求极致轻量和加载速度

什么时候应该上框架

  • ✗ 复杂单页应用（SPA）：大量状态管理、路由切换（如后台管理系统）
  • ✗ 超大型团队协作：组件复用率极高，需要类型约束（TypeScript）
  • ✗ 实时交互密集型：在线文档协作、即时通讯、复杂数据可视化
关键结论

落后不落后，不看技术栈，看的是能不能稳定交付、长期维护、快速迭代。原生三件套在这三点上，依然是最强的。

HTML + CSS 配合的三步工作流

  1. 写语义 HTML：用正确的标签（article, nav, time, figure 等），不加 class 也能读通结构。
  2. 加 BEM class：组件块（Block）、元素（Element）、修饰符（Modifier）。
  3. CSS 只引用变量：var(--color-primary)，不写死任何值。
/* 错误的顺序：先想样式，再套结构 */
.div-box { width: 300px; background: white; }
<div class="div-box">内容</div>

/* 正确的顺序：先语义，再组件，再变量 */
<article class="card">
  <h2 class="card__title">标题</h2>
</article>

.card {
  background: var(--color-surface);
  border-radius: var(--radius-lg);
}

可直接套用的页面模板

<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>页面标题</title>
  <!-- 1. 主题变量和基础样式 -->
  <link rel="stylesheet" href="components/base/variables.css">
  <link rel="stylesheet" href="components/base/reset.css">
  <link rel="stylesheet" href="components/base/typography.css">
  <!-- 2. 布局与组件 -->
  <link rel="stylesheet" href="components/layout/container.css">
  <link rel="stylesheet" href="components/layout/grid.css">
  <link rel="stylesheet" href="components/components/button.css">
  <link rel="stylesheet" href="components/components/card.css">
</head>
<body>

  <!-- 3. 语义化结构 -->
  <header class="site-header">
    <nav class="nav container">
      <a class="nav__logo" href="/">Logo</a>
      <ul class="nav__menu">
        <li><a class="nav__link" href="#about">关于</a></li>
        <li><a class="nav__link" href="#contact">联系</a></li>
      </ul>
    </nav>
  </header>

  <main class="container">
    <article class="card">
      <h1 class="card__title">文章标题</h1>
      <p class="card__text">使用 var(--color-text) 自动适配暗色主题。</p>
      <div class="card__actions">
        <button class="btn btn--primary">确认</button>
        <button class="btn btn--ghost">取消</button>
      </div>
    </article>
  </main>

  <footer class="site-footer">
    <p>© 2026</p>
  </footer>

  <!-- 4. 交互逻辑 -->
  <script src="demo.js"></script>
</body>
</html>
团队纪律

所有成员在写 HTML 前，先确认 CSS 变量表（variables.css）中是否已有对应令牌。如果没有，先在变量表中注册，再在 HTML/CSS 中使用。禁止私自创造新的颜色或间距值。
