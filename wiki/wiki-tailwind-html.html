什么是 tailwind-html

tailwind-html/ 目录下收录了 704 个纯 HTML 组件片段，全部使用 Tailwind CSS 的 Utility Class 编写。这些组件来源于 Tailwind UI 的设计体系，覆盖后台管理、电商、营销三类场景，可直接复制到项目中使用。

核心定位

tailwind-html 是即拿即用的 HTML 片段库。没有构建步骤，没有 JS 依赖，不需要 npm install。找到需要的组件，复制 HTML 代码，粘贴到你的页面中即可渲染。

与 Tailwind CSS 的关系

tailwind-html 中的组件假设你的项目已经引入了 Tailwind CSS。每个 HTML 文件只包含结构代码和 Tailwind 类名，不包含任何自定义 CSS 或 JavaScript。

<!-- 典型结构：div + Tailwind classes -->
<div class="bg-white rounded-lg shadow p-6">
  <h2 class="text-lg font-semibold text-gray-900">标题</h2>
  <p class="mt-2 text-sm text-gray-600">描述文字</p>
</div>

技术特征

特征说明
纯 Utility Class100% 使用 Tailwind 原子类，无自定义 CSS
语义化 HTML使用正确的标签（table、nav、form、aria-* 等）
响应式设计大量使用 sm: / md: / lg: / xl: 断点前缀
暗黑模式支持部分组件提供 "on dark" 变体
SVG 图标使用内联 SVG，不依赖外部图标库
无 JS 依赖纯 HTML + CSS，交互逻辑需自行实现
Tailwind UI 图源图片使用 tailwindui.com/plus-assets 域名

如何使用

前置条件

使用这些组件前，确保你的项目已引入 Tailwind CSS：

<!-- 方式 1：CDN（仅开发测试） -->
<script src="https://cdn.tailwindcss.com"></script>

<!-- 方式 2：构建后引入（生产推荐） -->
<link rel="stylesheet" href="dist/output.css">

使用步骤

  1. 浏览目录找到需要的组件类型（如 marketing/sections/heroes/）
  2. 打开 HTML 文件，复制全部代码
  3. 粘贴到你的页面中
  4. 替换占位文字和图片 URL
  5. 根据品牌调整颜色（将 indigo/gray 替换为你的品牌色）

选择建议

如果你的项目已经在使用 Tailwind CSS，tailwind-html 是最自然的组件来源。如果你的项目使用本项目的原生 CSS 体系，建议继续使用 components/ 目录下的组件，风格更统一。

实用技巧

1. 批量替换品牌色

Tailwind 组件默认使用 indigo 作为主色调。替换品牌色的最简单方法是使用编辑器的查找替换：

# 将 indigo 替换为你的品牌色
indigo → blue    # 蓝色品牌
indigo → emerald # 绿色品牌
indigo → rose    # 红色品牌

2. 提取 @apply 组件

如果同一个样式组合频繁出现，可以提取为 CSS 组件：

/* 在你的 CSS 文件中 */
.btn-primary {
  @apply rounded-md bg-indigo-600 px-3 py-2 text-sm font-semibold
         text-white shadow-xs hover:bg-indigo-500
         focus-visible:outline-2 focus-visible:outline-offset-2
         focus-visible:outline-indigo-600;
}

3. 与本项目 CSS 变量结合

可以在 tailwind.config.js 中映射本项目的 CSS 变量，实现两套系统共存：

// tailwind.config.js
module.exports = {
  theme: {
    extend: {
      colors: {
        primary: {
          DEFAULT: 'var(--primary)',
          foreground: 'var(--primary-foreground)',
        },
      },
    },
  },
}

4. 移动端优先

Tailwind 采用移动优先的响应式策略。理解这个规则有助于快速修改布局：

<!-- 默认是移动端样式，sm/md/lg 是更大的屏幕 -->
<div class="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3">
  <!-- 手机 1 列 → 平板 2 列 → 桌面 3 列 -->
</div>
