骨架总览

本项目中的所有 Wiki 页面（如 theme-wiki.html、wiki-html-guide.html 等）共享同一套 HTML 骨架。这套骨架是单文件、零依赖、自包含的——所有 CSS 和 JS 都内嵌在同一个 HTML 文件中，不需要任何外部构建工具或框架。

核心特征
  • 单文件：一个 .html 文件即可运行，无 node_modules，无构建步骤
  • CSS 变量驱动：颜色、间距、圆角全部通过 :root 变量定义，一处修改全局生效
  • 暗色主题内置：通过 [data-theme="dark"] 覆盖变量，JS 一键切换
  • 响应式侧边栏：桌面端固定 260px 侧边导航，移动端自动收起为汉堡菜单
  • 滚动高亮：滚动主内容区时，侧边栏自动高亮当前章节

三层结构

从宏观上看，每个 Wiki 页面由三层组成：

层级内容作用
Head<meta> + <style>定义字符集、视口、CSS 变量、全部组件样式
Body.sidebar + .main侧边栏导航 + 主内容阅读区
Script<script>（闭合标签前）主题切换、滚动高亮、移动端菜单控制
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <!-- 1. Head：meta + 全部 CSS -->
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>页面标题</title>
  <style>
    :root { /* CSS 变量 */ }
    [data-theme="dark"] { /* 暗色覆盖 */ }
    /* 布局 + 组件样式 */
  </style>
</head>
<body>
  <!-- 2. Body：汉堡按钮 + 布局容器 -->
  <button class="menu-toggle">☰</button>
  <div class="wiki-layout">
    <aside class="sidebar">...</aside>
    <main class="main">...</main>
  </div>

  <!-- 3. Script：交互逻辑 -->
  <script>
    /* 主题切换 + 滚动高亮 */
  </script>
</body>
</html>

Head 与 CSS 变量系统

Head 内只有三个元素：<meta charset>、<meta viewport>、<title>，以及一个巨大的 <style> 块。

CSS 变量定义顺序

<style>
  /* 第 1 段：亮色主题变量 */
  :root {
    --background: #ffffff;
    --foreground: #09090b;
    --primary: #18181b;
    --secondary: #f4f4f5;
    --muted: #f4f4f5;
    --border: #e4e4e7;
    --radius: 0.75rem;
    --success: #10b981;
    --warning: #f59e0b;
    --danger: #ef4444;
    --sidebar-width: 260px;
    /* ... 以及透明度阶梯变量 --primary-5, --primary-8 等 */
  }

  /* 第 2 段：暗色主题覆盖 */
  [data-theme="dark"] {
    --background: #09090b;
    --foreground: #fafafa;
    --primary: #fafafa;
    --border: rgba(255,255,255,0.10);
    /* ... 只覆盖需要变的变量 */
  }

  /* 第 3 段：基础重置 */
  * { box-sizing: border-box; margin: 0; padding: 0; }
  body { font-family: ...; background: var(--background); color: var(--foreground); }

  /* 第 4 段：布局与组件样式 */
  .wiki-layout { display: flex; min-height: 100vh; }
  .sidebar { ... }
  .main { ... }
  /* ... callout、table、code 等 */
</style>
不要改错顺序

[data-theme="dark"] 必须放在 :root 之后，否则覆盖不生效。基础重置（*、body）必须在变量定义之后，否则 var(--background) 还不可用。

透明度变量的命名规律

为了在做提示框背景、悬停态时不用写死 rgba，项目使用了一套数字后缀变量：

--primary-5   → rgba(24,24,27,0.05)   /* 极淡背景 */
--primary-8   → rgba(24,24,27,0.08)   /* 悬停态背景 */
--primary-12  → rgba(24,24,27,0.12)   /* 激活态背景 */
--primary-15  → rgba(24,24,27,0.15)   /* 边框 */
--primary-20  → rgba(24,24,27,0.20)   /* 深边框 */
--primary-30  → rgba(24,24,27,0.30)   /* 弱化文字 */

暗色主题下，这些变量会自动切换为 rgba(250,250,250, ...)，确保对比度始终舒适。

Body 布局骨架

Body 内部只有三个顶层元素：

  1. 汉堡按钮：<button class="menu-toggle">，只在移动端显示（@media (max-width: 900px) 控制）
  2. 布局容器：<div class="wiki-layout">，flex 横向排列
  3. 脚本块：<script>，放在 </body> 闭合前
<body>
  <!-- 移动端菜单开关 -->
  <button class="menu-toggle"
    onclick="document.querySelector('.sidebar').classList.toggle('open')">
    ☰
  </button>

  <!--  flex 容器：sidebar + main -->
  <div class="wiki-layout">
    <aside class="sidebar">...</aside>
    <main class="main">...</main>
  </div>

  <script>...</script>
</body>

侧边栏导航

侧边栏是一个固定定位的 <aside>，内部包含四个固定部分：

部分标签作用
页面标题<h1>Wiki 页面名称，如 "Wiki 构建指南"
版本/副标题<span class="version">简短描述，如 "HTML 骨架与手动搭建"
主题切换<button class="theme-toggle">🌙/☀️ 图标 + "暗色模式" 文字
导航菜单<nav>按 .nav-group 分组，内部是 <ul><li><a>

导航链接分组规范

<nav>
  <div class="nav-group">
    <div class="nav-group-title">概览</div>
    <ul>
      <li><a href="#overview" class="active">骨架总览</a></li>
      <li><a href="#structure">三层结构</a></li>
    </ul>
  </div>
  <div class="nav-group">
    <div class="nav-group-title">Wiki 导航</div>
    <ul>
      <li><a href="../index.html">首页</a></li>
      <li><a href="theme-wiki.html">主题定制 Wiki</a></li>
      <!-- 其他 Wiki 页面 -->
    </ul>
  </div>
</nav>
关键点
  • 锚点链接（如 href="#overview"）对应主内容区 <h2 id="overview">
  • 当前激活项加 class="active"，由 JS 滚动监听自动切换
  • "Wiki 导航"分组是必选项，用于在所有 Wiki 页面之间互相跳转

主内容区

主内容区是一个 <main class="main">，内部直接放各种内容元素。Wiki 系统内置了以下排版组件：

元素Class / 标签用途
章节标题<h2 id="xxx">必须带 id，用于锚点定位和滚动高亮
小节标题<h3>子章节，不带底边框
代码行<code>行内代码，自动带背景和圆角
代码块<pre><code>深色背景块，带横向滚动条
提示框.callout普通提示；加 .success / .warning / .danger 变颜色
表格<table>全宽、带边框、表头灰色背景
标签.tag小徽章，用于标注状态或分类
步骤条.steps > li自动编号（1、2、3…）的列表

JS 交互功能

每个 Wiki 页面末尾都有同一段脚本，负责三件事：

1. 主题切换

function applyTheme(theme) {
  if (theme === 'dark') {
    html.setAttribute('data-theme', 'dark');
    icon.textContent = '☀️';
    label.textContent = '亮色模式';
  } else {
    html.removeAttribute('data-theme');
    icon.textContent = '🌙';
    label.textContent = '暗色模式';
  }
}
function toggleTheme() {
  const current = html.getAttribute('data-theme') === 'dark' ? 'dark' : 'light';
  const next = current === 'dark' ? 'light' : 'dark';
  applyTheme(next);
  localStorage.setItem('theme', next);
}
// 初始化：优先读 localStorage，其次读系统偏好
const saved = localStorage.getItem('theme');
const prefersDark = window.matchMedia('(prefers-color-scheme: dark)').matches;
applyTheme(saved || (prefersDark ? 'dark' : 'light'));

2. 滚动高亮当前章节

const links = document.querySelectorAll('.sidebar nav a');
const sections = document.querySelectorAll('.main h2[id]');

function onScroll() {
  let current = '';
  sections.forEach(sec => {
    if (sec.getBoundingClientRect().top <= 120) current = sec.getAttribute('id');
  });
  links.forEach(link => {
    link.classList.toggle('active', link.getAttribute('href') === '#' + current);
  });
}
window.addEventListener('scroll', onScroll);
onScroll();

3. 移动端点击链接后收起侧边栏

links.forEach(link => {
  link.addEventListener('click', () => {
    document.querySelector('.sidebar').classList.remove('open');
  });
});
JS 是通用的

这段脚本在所有 Wiki 页面中完全一致，不需要根据内容修改。复制粘贴即可。

四步构建法

要手动创建一个新的 Wiki 页面，只需四步：

  1. 复制任意现有 Wiki 文件（如 wiki-html-guide.html），重命名为你的新文件名（如 wiki-new-topic.html）
  2. 修改三个标识位：<title>、.sidebar h1、.sidebar .version
  3. 重写导航和内容：在 .sidebar nav 中定义章节锚点，在 .main 中写入正文，确保 h2 id 与 a href 一一对应
  4. 更新所有 Wiki 文件的"Wiki 导航"分组：在每个现有 Wiki 页面和 index.html 的侧边栏中加入新页面链接

检查清单

发布新 Wiki 页面前，逐项确认：

  • <html lang="zh-CN"> 语言属性正确
