Tailwind CSS 是什么

Tailwind 是一个 Utility-First 的 CSS 框架。它不给你预制的 `.btn` 或 `.card`，而是给你数百个原子级 class，让你像搭积木一样组合出界面。

<!-- Bootstrap 思维：给预制组件 -->
<button class="btn btn-primary btn-lg">提交</button>

<!-- Tailwind 思维：给原子工具 -->
<button class="bg-blue-600 text-white px-6 py-2 rounded-lg font-bold hover:bg-blue-500">
  提交
</button>

核心特点

特性说明对你的影响
Utility-First每个 class 只做一件事HTML 变长，但 CSS 文件极小
Purge 机制只打包用到的 class生产 CSS 通常 < 10KB
配置驱动颜色、间距在 tailwind.config.js 定义一处改色，全站跟随
无预设立场不强制任何视觉风格从零设计，自由度极高
JIT 编译即时生成任意值 classw-[123px] 随手写

与现有原生组件库对比

你现在已经有了三套原生 CSS 系统。Tailwind 和它们不是竞争关系，而是不同层级的工具。

维度你的原生库 components/Tailwind CSS
开发速度复制组件 HTML，改内容即用从零拼 class，前期慢后期快
文件体积全量加载（可模块化）Purge 后极小
定制自由度受组件预设约束完全自由，无预设
学习成本低（看文档即可用）中高（需记忆 class 名）
AI 友好度极高（AI 理解 BEM 很准）中（AI 常遗漏 class）
构建依赖零依赖，直接引用需要 Node.js + 构建步骤
WordPress直接粘贴 CSS 即可需先构建再上传
暗色主题手动写 [data-theme] 覆盖dark: 前缀一行搞定
关键结论

Tailwind 的真正优势不是"写得更爽"，而是构建时 tree-shaking 带来的极致体积控制。如果你的项目 CSS 已经超过 50KB，Tailwind 能带来明显收益。否则，原生库更简单可靠。

什么时候用哪个

项目类型？
│
├── 纯 HTML 页面 / WordPress / 无构建环境？
│     └── 用原生库 components/ + huahao-blog/ + nichehub/
│         → 零依赖，保存即刷新，直接粘贴进 WP
│
├── React / Vue / 大型 SPA？
│     └── 用 Tailwind
│         → 组件级隔离，tree-shaking，JS 生态最佳拍档
│
├── 混合项目（既有 CMS 又有自定义页面）？
│     └── 混合策略（见下文）
│         → CMS 内容用原生库，自定义界面用 Tailwind
│
└── 追求极致加载速度（落地页 / 营销页）？
      └── 用 Tailwind（Purge 后 < 10KB）
          → 比加载 Bootstrap 或全量组件库快 3-5 倍

混合策略：Tailwind + 原生库共存

你不需要二选一。最务实的做法是分层使用：

层级技术负责什么
基础变量原生 CSS 变量颜色、间距、字体定义
布局TailwindFlexbox、Grid、间距
组件外壳Tailwind边框、圆角、阴影、背景
组件内核原生组件 CSS交互状态、动画、复杂选择器
内容块huahao / nichehub文章内富媒体组件

示例：混合写一个卡片

<!-- Tailwind 负责布局外壳 -->
<div class="bg-white rounded-xl shadow-md p-6 max-w-sm">
  
  <!-- 原生库负责内容排版 -->
  <h3 class="card__title">产品标题</h3>
  <p class="card__text">描述文字...</p>
  
  <!-- Tailwind 负责按钮布局 -->
  <div class="flex gap-2 mt-4">
    <button class="btn btn--primary">购买</button>
    <button class="btn btn--ghost">详情</button>
  </div>
</div>
规则

Tailwind 管位置、大小、间距、颜色；原生组件管行为、状态、语义。两者不要同时控制同一属性（如同时定义 padding）。

设计令牌映射

如果你决定引入 Tailwind，第一步是把现有的 CSS 变量映射到 tailwind.config.js，确保两套系统颜色一致。

通用库 → Tailwind

// tailwind.config.js
module.exports = {
  theme: {
    extend: {
      colors: {
        primary: {
          DEFAULT: '#2563eb',
          dark: '#1d4ed8',
          light: '#eff6ff',
        },
        danger: '#dc2626',
        success: '#059669',
        warning: '#d97706',
        surface: '#ffffff',
        'text-main': '#1e293b',
        'text-light': '#64748b',
      },
      spacing: {
        '4px': '0.25rem',
        '8px': '0.5rem',
        '12px': '0.75rem',
        '16px': '1rem',
        '24px': '1.5rem',
        '32px': '2rem',
      },
      borderRadius: {
        'sm': '4px',
        'md': '8px',
        'lg': '12px',
      },
      fontFamily: {
        sans: ['-apple-system', 'BlinkMacSystemFont', 'Segoe UI', 'Roboto', 'sans-serif'],
        mono: ['SF Mono', 'Monaco', 'Consolas', 'monospace'],
      },
    },
  },
}

好处

一旦映射完成，你可以在 Tailwind 中写 text-primary、bg-nh-secondary，它会输出和你原生变量完全一致的颜色。两套系统颜色同源，不会出现"两个蓝色"。

WordPress 中使用 Tailwind

Tailwind 需要构建步骤，这在 WordPress 中比较麻烦。以下是三种可行方案：

方案 A：预构建 + 上传（推荐）

  1. 本地创建项目，安装 Tailwind：npm install -D tailwindcss
  2. 配置 tailwind.config.js，映射你的设计令牌
  3. 写一个 input.css，只 @tailwind 你需要的层
  4. 运行构建：npx tailwindcss -i input.css -o output.css --minify
  5. 将 output.css 上传到 WP 子主题目录
  6. 在 functions.php 中 wp_enqueue_style 引入

方案 B：CDN 快速原型（不用于生产）

<!-- 仅用于测试，体积大且无 Purge -->
<script src="https://cdn.tailwindcss.com"></script>
<script>
  tailwind.config = {
    theme: {
      extend: {
        colors: { primary: '#2563eb' }
      }
    }
  }
</script>
警告

CDN 版加载 300KB+ 的未压缩 CSS，没有 Purge 机制。绝对不要用于生产环境。

方案 C：Tailwind + 原生库混合（最佳实践）

对于既有 CMS 内容又有自定义主题的 WP 站点：

  1. CMS 生成的内容（文章、页面）使用原生库 hh-wordpress-bundle.css，保证编辑器内的 HTML 块正常显示
  2. 自定义主题部分（导航、页脚、特殊 landing page）使用预构建的 Tailwind CSS
  3. 两套 CSS 同时加载，通过特异性或 !important 策略避免冲突

三种引入方式详解

方式 1：CLI 构建（最简单）

# 1. 初始化
npm init -y
npm install -D tailwindcss
npx tailwindcss init

# 2. 编辑 tailwind.config.js
module.exports = {
  content: ["./src/**/*.{html,js}"],
  theme: { extend: {} },
  plugins: [],
}

# 3. 创建 input.css
@tailwind base;
@tailwind components;
@tailwind utilities;

# 4. 构建
npx tailwindcss -i input.css -o dist/output.css --watch

方式 2：Vite 集成（现代前端）

# 安装
npm install -D tailwindcss postcss autoprefixer
npx tailwindcss init -p

# vite.config.js
import { defineConfig } from 'vite'
export default defineConfig({
  css: { postcss: './postcss.config.js' }
})

# 在 CSS 中直接写
@import 'tailwindcss/base';
@import 'tailwindcss/components';
@import 'tailwindcss/utilities';

方式 3：单文件 HTML（无构建）

<!-- 使用 Tailwind CLI 预构建后，直接引用 -->
<link rel="stylesheet" href="dist/output.css">

<!-- 你的 HTML 直接用 Tailwind class -->
<div class="flex items-center justify-center min-h-screen bg-gray-100">
  <div class="bg-white p-8 rounded-lg shadow-lg">
    <h1 class="text-2xl font-bold text-gray-800">Hello</h1>
  </div>
</div>

用 Tailwind 写组件

Tailwind 推荐用 @apply 提取组件，而不是在 HTML 里写 20 个 class。

错误示范：HTML 膨胀

<!-- 不要这样做：HTML 难以阅读和维护 -->
<button class="inline-flex items-center justify-center gap-2 px-4 py-2 
  bg-blue-600 text-white text-sm font-medium rounded-md border 
  border-transparent hover:bg-blue-500 focus:outline-none 
  focus:ring-2 focus:ring-blue-500 focus:ring-offset-2 
  disabled:opacity-50 disabled:cursor-not-allowed">
  提交
</button>

正确做法：@apply 提取

/* components.css */
.btn {
  @apply inline-flex items-center justify-center gap-2 px-4 py-2
         text-sm font-medium rounded-md border border-transparent
         focus:outline-none focus:ring-2 focus:ring-offset-2
         disabled:opacity-50 disabled:cursor-not-allowed;
}
.btn--primary {
  @apply bg-blue-600 text-white hover:bg-blue-500
         focus:ring-blue-500;
}
.btn--ghost {
  @apply bg-transparent text-blue-600 border-blue-600
         hover:bg-blue-50;
}
<!-- HTML 简洁清晰 -->
<button class="btn btn--primary">提交</button>
<button class="btn btn--ghost">取消</button>
最佳实践

用 @apply 提取的组件，和你原生库的 BEM 命名完全一致。这样既享受了 Tailwind 的变量系统，又保持了 HTML 的可读性。AI 生成代码时也更准确。

迁移策略

如果你已经有了 components/ 或 huahao-blog/，不需要重写。渐进式迁移即可。

阶段行动产出
第 1 周引入 Tailwind CDN（仅本地原型）验证组件用 Tailwind 重写后的效果
第 2 周配置 tailwind.config.js 映射现有变量颜色、间距与原生库 100% 一致
第 3 周新页面用 Tailwind，旧页面保持原生两套系统并行，零冲突
第 4 周把高频组件用 @apply 提取HTML 恢复简洁，保留 Tailwind 优势
第 2 月逐步替换旧页面的布局代码体积下降，速度提升
不要做的事
  • 不要一次性重写所有页面——风险太高
  • 不要把 Tailwind 和 Bootstrap 混用——冲突地狱
  • 不要在 WordPress 生产环境用 CDN 版——体积爆炸
  • 不要放弃原生库去追 Tailwind——原生库在 CMS 场景更优

AI 协作提示词

使用 AI 生成 Tailwind 代码时，需要比原生 CSS 更详细的约束。

标准约束模板

Tailwind 项目环境：
1. 使用 Tailwind CSS v3，配置中已定义颜色：primary=#2563eb, danger=#dc2626, success=#059669
2. 暗色主题用 dark: 前缀，不要手动写 @media (prefers-color-scheme)
3. 响应式断点：sm=640px, md=768px, lg=1024px, xl=1280px
4. 组件提取用 @apply，不要在 HTML 里写超过 5 个 class
5. 间距使用 4 的倍数：p-4, m-6, gap-2（对应 16px, 24px, 8px）
6. 字体使用 font-sans（系统字体栈），不要写死具体字体名
7. 圆角使用 rounded-md（8px）作为默认，rounded-lg（12px）用于卡片
8. 输出完整的 HTML + CSS 代码块

场景示例：用 Tailwind 重写按钮组件

请用 Tailwind CSS 重写以下按钮组件，要求：

原代码：
.btn { padding: 8px 16px; background: var(--color-primary); color: white; }

要求：
- 使用 @apply 提取，保留 .btn .btn--primary .btn--ghost .btn--danger 命名
- 支持 hover、focus、disabled 状态
- 暗色模式下主色反转为浅蓝色
- HTML 示例展示三种按钮并排
最终建议

Tailwind 不是银弹，它是体积敏感型项目的优化工具。如果你的项目已经用原生库跑得很好，不要为了用 Tailwind 而重写。但当你的 CSS 超过 50KB、需要暗色主题、或在使用 React/Vue 时，引入 Tailwind 的收益会非常明显。
