整体架构

你的项目现在由 三个独立项目 + 一个模式层 组成，共享同一套分层哲学，但服务于不同场景。

该用哪一层？

拿到一个新需求时，按以下决策树选择：

我要做什么？
│
├── 搭建一个全新的网站界面？
│     └── 用 components/（按钮、卡片、导航、弹窗…）
│
├── 写一篇 WordPress 博客文章，需要丰富的内容组件？
│     ├── 制造业 / B2B / 工厂主题？
│     │     └── 用 huahao-blog/（对比矩阵、评分卡、流程图…）
│     │
│     └── 刀具 / 厨具 / 产品评测主题？
│           └── 用 nichehub/（知识卡片、Steel Finder、对比条…）
│
├── 第三个项目也需要对比矩阵 / 决策树 / 评分卡？
│     └── 用 patterns/（通用命名，适配任何品牌色）
│
└── 只改颜色不改结构？
      └── 改对应项目的 variables.css，组件自动跟随
场景选哪个原因
企业官网、落地页、后台系统components需要完整的界面原子：按钮、表单、表格、弹窗
无纺布/制造业博客文章huahao已有46个B2B内容块，直接复制HTML进WordPress
刀具/厨具知识中心nichehub专为Steel Finder、材料对比、工艺路线设计
评测网站、导购页面patterns提取的通用模式，换色即可复用

通用组件库 components/

用于搭建任何网站的界面层。设计理念是"原子化 + 变量驱动"。

加载顺序（不可变）

<!-- 1. 基础层：必须先加载 -->
<link rel="stylesheet" href="components/base/variables.css">
<link rel="stylesheet" href="components/base/reset.css">
<link rel="stylesheet" href="components/base/typography.css">

<!-- 2. 布局层 -->
<link rel="stylesheet" href="components/layout/container.css">
<link rel="stylesheet" href="components/layout/grid.css">

<!-- 3. 按需加载组件层 -->
<link rel="stylesheet" href="components/components/button.css">
<link rel="stylesheet" href="components/components/card.css">
<link rel="stylesheet" href="components/components/form.css">
<!-- ...其他组件 -->
纪律

variables.css 必须在第一位。它定义了所有颜色和间距令牌，后续组件都依赖这些变量。没有它，组件会显示为黑白色。

修改主题色

打开 components/base/variables.css，找到以下行修改：

:root {
  --color-primary: #2563eb;        /* 改这里 → 主品牌色 */
  --color-success: #059669;        /* 改这里 → 成功色 */
  --color-danger: #dc2626;         /* 改这里 → 危险色 */
  --font-sans: "Inter", sans-serif; /* 改这里 → 正文字体 */
}

Huahao 博客库 huahao-blog/

专为 WordPress 博客文章设计。46个内容组件，全部使用 hh- 前缀，避免与主题冲突。

WordPress 使用（推荐）

  1. 打开 huahao-blog/hh-wordpress-bundle.css
  2. 全选复制（约 84KB）
  3. 进入 WP 后台 → 外观 → 自定义 → 额外 CSS
  4. 粘贴 → 点击发布
  5. 在文章编辑器中添加「自定义 HTML」块，粘贴组件 HTML

获取组件 HTML

打开 huahao-blog/index.html（本地双击即可），找到你想用的组件，复制对应的 HTML 代码。例如：

<!-- 对比矩阵 -->
<div class="hh-matrix">
  <div class="hh-matrix-header">...</div>
  <div class="hh-matrix-row">...</div>
</div>

<!-- 评分卡 -->
<div class="hh-scorecards">
  <div class="hh-scorecard">...</div>
</div>

修改品牌色

Huahao 使用灰度极简直线框风格。如需调整，编辑 huahao-blog/hh-variables.css：

:root {
  --hh-ink: #111827;        /* 主标题色 */
  --hh-text: #374151;       /* 正文色 */
  --hh-border: #e5e7eb;     /* 边框色 */
  --hh-font-head: Georgia, serif;  /* 标题字体 */
}
优势

Huahao 组件全部使用 纯 CSS，无 JS 依赖。手风琴用 <details><summary>，进度条用 div 宽度，条形图用 grid 布局。粘贴进 WordPress 即可生效，不需要安装任何插件。

NicheHub 知识中心 nichehub/

专为 刀具/厨具知识型网站设计。包含 Steel Finder、材料对比、工艺路线等垂直领域组件。

本地开发

双击 nichehub/index.html 即可查看完整的 LeeKnives Knowledge Center 页面。所有 CSS 已改为模块化引用：

<link rel="stylesheet" href="nh-variables.css"/>
<link rel="stylesheet" href="nh-base.css"/>
<link rel="stylesheet" href="nh-components.css"/>
<link rel="stylesheet" href="nh-responsive.css"/>

WordPress 使用

  1. 打开 nichehub/nichehub-bundle.css
  2. 全选复制（约 129KB）
  3. 进入 WP → 外观 → 自定义 → 额外 CSS → 粘贴 → 发布

修改品牌色

NicheHub 默认使用黑红配色（#1a1a1a + #E53E3E）。编辑 nichehub/nh-variables.css：

:root {
  --primary: #1a1a1a;          /* 主背景/文字 */
  --secondary: #E53E3E;        /* 强调色 → 改成你的品牌色 */
  --secondary-hover: #C53030;  /* 悬停色 */
  --text: #333333;             /* 正文色 */
}

可复用模式 patterns/

从 huahao-blog 中提取的 通用内容组件，已替换为语义化命名，可适配任何品牌。

使用方式

<!-- 1. 加载你的基础变量 -->
<link rel="stylesheet" href="components/base/variables.css">

<!-- 2. 加载需要的模式 -->
<link rel="stylesheet" href="patterns/comparison-matrix.css">
<link rel="stylesheet" href="patterns/score-card.css">

<!-- 3. 写 HTML -->
<div class="matrix">...</div>
<div class="score-cards">...</div>

变量约定

模式使用通用变量名并带有 fallback。只要你的项目定义了 --color-text、--color-border、--color-primary 等变量，模式就会自动适配。如果没有定义，也会用默认灰度色正常显示。

WordPress 集成指南

方式一：额外 CSS（适合单站）

最简单的方式，无需 FTP 或子主题。

  1. 打开对应项目的 *-bundle.css
  2. 全选复制
  3. WP 后台 → 外观 → 自定义 → 额外 CSS
  4. 粘贴 → 发布

方式二：子主题（适合长期维护）

如果样式量大、需要版本控制，使用子主题：

/* 在子主题 functions.php 中 */
add_action('wp_enqueue_scripts', function () {
    wp_enqueue_style('nichehub-style',
        get_stylesheet_directory_uri() . '/nichehub/nichehub-bundle.css',
        [], '1.0'
    );
});

方式三：单篇文章（适合一次性）

只在特定文章使用组件时，在文章顶部添加「自定义 HTML」块：

<style>
  /* 只粘贴该文章需要的组件CSS */
</style>
性能注意

额外 CSS 会加载在每一页。如果只在博客文章使用，而首页不用，建议用方式二（子主题条件加载）或方式三（单篇加载）。

新建项目模板

无论你用哪个项目，标准 HTML 骨架如下：

<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>页面标题</title>

  <!-- 基础层（必须） -->
  <link rel="stylesheet" href="components/base/variables.css">
  <link rel="stylesheet" href="components/base/reset.css">
  <link rel="stylesheet" href="components/base/typography.css">

  <!-- 布局层 -->
  <link rel="stylesheet" href="components/layout/container.css">
  <link rel="stylesheet" href="components/layout/grid.css">

  <!-- 通用组件层（按需） -->
  <link rel="stylesheet" href="components/components/button.css">
  <link rel="stylesheet" href="components/components/card.css">

  <!-- 业务组件层（按需） -->
  <!-- <link rel="stylesheet" href="patterns/comparison-matrix.css"> -->
  <!-- <link rel="stylesheet" href="patterns/score-card.css"> -->
</head>
<body>
  <main class="container">
    <!-- 你的内容 -->
  </main>
</body>
</html>

AI 协作提示词

使用 AI 生成代码时，在提示词中附加以下约束，确保生成的代码与组件库兼容：

标准约束模板 
项目环境约束：
1. 本项目使用原生 CSS 变量系统，禁止写死色值。
2. 通用组件使用 components/base/variables.css 中的变量。
3. Huahao 博客组件使用 hh- 前缀，NicheHub 组件不使用前缀。
4. 按钮用 .btn 类，卡片用 .card 类，禁止自己发明新命名。
5. 响应式断点：640px（手机）、768px（平板）、1024px（桌面）。
6. 输出完整的 HTML + CSS 代码块，不要省略。

分场景 Prompt 示例

场景 A：为 Huahao 博客添加新组件

请为 Huahao 博客添加一个时间线组件，用于展示订单生产流程。

环境：
- 使用 hh- 前缀命名
- 颜色引用 var(--hh-border)、var(--hh-ink)、var(--hh-muted)
- 字体引用 var(--hh-font-head)、var(--hh-font-mono)
- 参考现有 .hh-timeline 的结构风格

要求：
- 垂直时间线，左侧圆点 + 右侧内容卡片
- 支持已完成、进行中、未开始三种状态
- 纯 CSS，不使用 JavaScript

场景 B：为 NicheHub 修改品牌色

请将 NicheHub 的强调色从红色 (#E53E3E) 改为蓝色 (#2563eb)。

文件：nichehub/nh-variables.css

要求：
- 只修改 --secondary、--secondary-hover、--secondary-bright
- 保持所有其他变量不变
- 输出修改后的完整 :root 块

场景 C：从 patterns 引入对比矩阵到新项目

请写一个使用 patterns/comparison-matrix.css 的完整 HTML 示例。

环境：
- 已加载 components/base/variables.css（定义了 --color-primary 等）
- 已加载 patterns/comparison-matrix.css

内容：
- 比较三款刀具钢材：VG-10、AUS-8、D2
- 维度：硬度、防锈性、易磨性、价格
- 使用 .matrix 类结构
- 最佳项用 .matrix__best 标注
最终建议

把这套架构当作你的设计系统基础设施。不要每次新项目都从零写样式，而是从已有层中复制、修改、扩展。遇到好的新组件，回写到对应的项目中，形成复利效应。
